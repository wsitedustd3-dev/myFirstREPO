# 웹(Web) vs 앱(App) vs 웹앱(Web App) 완벽 비교

서비스의 목적과 구동 방식에 따른 핵심 차이점을 정리한 가이드입니다.

---

## 1. 웹 (Web / Website)
일반적으로 브라우저(Chrome, Safari 등)를 통해 접속하는 **정보 전달 중심**의 공간입니다.

* **특징:** 링크를 통해 페이지를 이동하며, 정적인 텍스트나 이미지를 보여주는 데 최적화되어 있습니다.
* **장점:** 별도의 설치가 필요 없고, 검색 엔진(Google, Naver)에 노출이 잘 됩니다.
* **예시:** 뉴스 사이트, 기업 소개 페이지, 블로그.

---

## 2. 앱 (App / Native App)
스마트폰의 스토어(App Store, Play Store)에서 **다운로드하여 설치**하는 프로그램입니다.

* **특징:** 기기(스마트폰)의 하드웨어 기능(카메라, GPS, 푸시 알림 등)을 100% 활용할 수 있어 성능이 매우 빠르고 부드럽습니다.
* **장점:** 오프라인에서도 실행 가능한 경우가 많고, 사용자 경험(UX)이 가장 뛰어납니다.
* **예시:** 인스타그램, 카카오톡, 모바일 게임.

---

## 3. 웹앱 (Web App)
웹의 기술을 사용하지만, 사용자에게는 **마치 앱처럼 느껴지게 만든** 사이트입니다.

* **특징:** 브라우저에서 실행되지만, 단순한 정보 전달을 넘어 채팅, 예약, 편집 등 **복잡한 기능**을 수행합니다. 최근에는 **PWA(Progressive Web App)** 기술을 통해 홈 화면 추가 및 푸시 알림도 가능해졌습니다.
* **장점:** 업데이트가 실시간으로 반영되며, 안드로이드와 아이폰 구분 없이 하나의 주소로 서비스할 수 있습니다.
* **예시:** 구글 문서(Docs), 노션(Notion), 웹 버전 유튜브.

---

## 📊 한눈에 비교하기

| 구분 | 웹 (Website) | 앱 (Native App) | 웹앱 (Web App) |
| :--- | :--- | :--- | :--- |
| **접속 방법** | 브라우저 주소창 입력 | 스토어에서 설치 | 브라우저 주소창 입력 |
| **개발 언어** | HTML, CSS, JS | Swift, Kotlin 등 | HTML, CSS, JS |
| **기능 활용** | 제한적 | 하드웨어 풀 활용 | 중간 정도 |
| **업데이트** | 실시간 (서버 반영) | 스토어 업데이트 필요 | 실시간 (서버 반영) |
| **속도** | 네트워크 의존적 | 매우 빠름 | 비교적 빠름 |

---

> **Tip:** 정보를 널리 퍼뜨리고 싶다면 **웹**, 최고의 성능이 필요하다면 **앱**, 설치 없이 앱의 기능을 제공하고 싶다면 **웹앱**을 선택하세요!
